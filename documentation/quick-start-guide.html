SeaWulf Quick Start Guide

The SeaWulf cluster is a collection of interconnected computers, known as nodes, designed to support computationally intensive tasks. Access to the SeaWulf cluster is restricted. All accounts must be created under official SeaWulf projects, which can only be requested by Stony Brook faculty. If you do not currently have access, please refer to the page:

Creating SeaWulf Accounts and Projects

Once your SeaWulf account has been created, the next step is to connect to the cluster. This can be done through an SSH connection or by using the Open OnDemand web interface. In this quick start guide, we'll focus on connecting via SSH.

All users connect to a set of login nodes, where they can design scripts and prepare jobs before submitting them to the compute nodes.

The login node you connect to determines which compute nodes you can access.

For the purposes of this guide, we'll connect to the Milan login nodes.

Mac/Linux

On Mac or Linux systems, you can connect using the terminal. Run the following command:

ssh -Y <NetID>@milan.seawulf.stonybrook.edu

The -Y flag enables X11 forwarding, which allows GUI applications to display on your local machine. If you're on a Mac, you'll need to have XQuartz installed. For more information, see:

Setting up X11 Forwarding

File System

When you log in, the pwd command will show that you're in your home directory. This is a personal 20GB space, backed up regularly, where you can store files, download software, and manage personal configurations. More details can be found in:

SeaWulf Filesystem

In addition to the home directory, you have access to a 20TB project scratch space located at:

/gpfs/scratch/<NetID>

This space is for temporary storage — files are automatically deleted after 30 days. For more information, refer to:

Scratch Space

Module System

While you can install software locally, much of what you need has already been installed on the cluster. To manage this large and diverse software environment, SeaWulf uses a module system.

This allows users to load only the software they need, depending on the architecture of the nodes they're working on.

You can check for available modules with:

module avail

To load a module:

module load <module-name>

We'll go over a sample use case in this guide.

Job Scheduling

Once you've set up your environment, the next step is to request compute resources. As mentioned earlier, login nodes are shared by many users and are not intended for running intensive jobs.

SeaWulf uses the SLURM job manager for scheduling work on compute nodes.

SLURM organizes the compute nodes into partitions (also called queues), each with its own constraints. For details, visit:

Node Types and Partitions

You can request an interactive session using the srun command. For example:

srun -p short-40core -N 1 -n 40 -t 1:00:00 --pty bash

Each flag has a specific meaning — see:

SLURM Basics

Alternatively, you can write a script using #SBATCH directives and submit it with the sbatch command. For example:

sbatch example.slurm

More info about script structure can be found here:

SLURM Scripts

Monitoring Jobs

After submitting a job, you can monitor it using:

squeue

This will show job status. If your job is marked PENDING, it's waiting for available resources. Wait time depends on job size, queue limits, and cluster usage.
